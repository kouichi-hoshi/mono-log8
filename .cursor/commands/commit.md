# /commit — コミット作成手順

目的: コミット前にlintを必ず実行し、日本語タイトルと箇条書きで修正内容を明確にする。

前提・制約:
- lint必須: `npm run lint` が失敗したらコミットしない。
- 機微情報・envファイルをコミットしない（AGENTS.mdの禁止事項を遵守）。
- 既存のコミットスタイルを尊重しつつ、以下のメッセージ形式を優先する。

手順:
1) 差分確認: `git status` / `git diff` で内容を確認する。
2) lint実行: `npm run lint`（失敗したら修正して再実行）。
3) コミットメッセージ作成（日本語）:
   - 1行目: タイトル（例: `UI: 認証モーダルの表示改善`）
   - 空行
   - 箇条書き: `- 変更点1` のように修正内容を個別に列挙（機能/ファイル/影響範囲が分かる粒度）。
4) ユーザー確認: タイトルと箇条書きを提示し、最終確認として `y/n` を尋ねる（`y` なら続行、`n` なら中止）。
5) コミット実行（例）:
```
git commit -m "$(cat <<'EOF'
UI: 認証モーダルの表示改善

- ログインボタン押下時のモーダル表示遅延を解消
- キャンセル操作後にフォーカスを元の入力へ戻す処理を追加
EOF
)"
```
6) 失敗時対応:
   - lintエラー: 対応後に再度 `npm run lint` → コミット。
   - 競合や差分不足: 差分を確認し、必要なら再度コミットを実行。
