# /test-quality — テスト品質チェック

このスラッシュコマンドは、`.codex/skills/test-quality/SKILL.md` に定義された手順に従って、テストそのものの品質をチェックします。

## 手順

以下のskillファイルを参照し、その手順に従って実行してください：

1. `.codex/skills/test-quality/SKILL.md` を読み込む
2. skillに定義された手順（Workflow）に従って実行する
3. テストの壊れにくさ、無駄の排除、意味の明確化、重複防止をチェックする
4. 設計へのフィードバックを検討する

## 注意事項

- テストが壊れにくい（flakyになりにくい）状態を維持する
- テストが無駄に多くならず、重複せず、意味のある仕様保証になるようにする
- テストが設計改善（境界/依存/責務）にフィードバックできる運用にする
