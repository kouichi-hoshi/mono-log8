# /doc-consistency — 仕様・設計・計画の整合性チェック

目的: /docs配下のファイルを相互参照し、矛盾や抜け漏れを検知する。修正案を提示し、ユーザー許可後に修正を行う

手順:
- 検知対象:
  - 要件定義にあるが設計/計画/チェックリストに無い、または内容が異なる。
  - 設計にある技術・実装要件が計画やチェックリストに反映されていない。
  - 作業計画とチェックリストのフェーズや完了条件が不一致。
- レポート:
  - 問題ごとに「影響範囲」「根拠（ファイルと要約）」「修正提案」を列挙。
  - 修正は必ずユーザーの許可を得てから実行する。
- 修正後:
  - 該当ドキュメントを更新したら、再度ドキュメントの更新箇所を突き合わせて矛盾が解消されたか確認。
